## Goal
- Fix production deployment errors (middleware crash, destinations 503/500, missing translations, auth, footer layout)

## Constraints & Preferences
- Vercel production at `www.akmleva.pt` and `beta-app-tau.vercel.app`
- GitHub Actions workflow runs deploy + migrate on push to `main`
- Neon PostgreSQL with pooler URL and unpooled URL
- Vercel CLI requires `--yes` for non-interactive production deploys
- `VERCEL_TOKEN` secret added to GitHub repo Secrets

## Progress
### Done
- **Prisma advisory lock timeout**: fixed workflow step to use `DATABASE_URL_UNPOOLED` secret with fallback to `DATABASE_URL`
- **`wv_destinations.categorias` column missing**: removed `categorias Json? @db.JsonB` from `prisma/schema.prisma`, removed from `destinationSelectForLookup`, `rowToDestination`, and `MockDestination` type; Prisma client regenerated
- **Duplicate translation keys**: renamed second `results.clearFilters` to `results.clearAll` in `en.json`, `es.json`, `fr.json`, `pt.json`; updated `ResultsPage.tsx` filter toolbar button to use `t('clearAll')`
- **AI insights 503 spam**: API route returns `{ ok: true, answer: null }` instead of 503 when `ML_SERVICE_BASE_URL` is missing; frontend debounced with 1.5s `setTimeout`
- **Orphaned submodules**: `Zoo-Code` and `google-maps-scraper` removed from git index (no `.gitmodules`); added to `.gitignore`
- **Vercel deploy in CI**: changed from `vercel --prod --token=${{ secrets.VERCEL_TOKEN }}` to `vercel --prod --yes` with `VERCEL_TOKEN` env var
- **Auth-edge crash**: changed `throw new Error('AUTH_SECRET is required...')` to `console.warn()` so middleware doesn't crash when `AUTH_SECRET` is missing in production
- **AUTH_SECRET set in Vercel**: generated base64 key and added via `vercel env add AUTH_SECRET production --sensitive`
- **Missing translation keys**: updated `DestinationsBrowsePage.tsx` to use existing keys — `t('loadError')` instead of `t('errorLoading')`, `t('resultsCount', { count: total })` instead of `t('results', { count: total })`; added `retry` key to all 4 locale files (`pt.json` "Tentar novamente", `en.json` "Retry", `es.json` "Reintentar", `fr.json` "Réessayer")
- **Footer full-width**: moved `<AppFooter />` outside the `max-w-7xl` container in both `AboutPage.tsx` (line 337→outside line 338) and `ContactPage.tsx` (line 316→outside line 317) so footer spans full width
- **Health endpoint**: created `/api/health` — returns `{"ok":true,"env":"not-set"}` with 200, confirms basic function execution works
- **Destinations API 500 fixed**: the root cause was `unsplash-cache.json` having mixed value types (3029 out of 33217 values are objects `{url,photographer,...}` instead of plain strings). The `lookupCachedUrl` function in `destination-image.ts` assumed all values were strings and called `.trim()` on objects, crashing with `"h?.trim is not a function"`. Fix: added `normalizeCacheValue()` helper that extracts the `url` property from object values. This affects ALL routes using `resolveDestinationImageUrl` (destinations, v1/destinations, results, slug, trip-recommendation, mock-travel/search).
- **Destinations route hardened**: changed from static top-level imports to `await import()` inside the handler with outer try-catch, so module-load failures don't cause empty-500 responses
- **Mock bundle fallback safety**: both legacy and v1 routes wrapped `loadMockTravelBundle()` in try-catch to return empty results if mock bundle file is unavailable/malformed
- **`TRAVEL_CATALOG_SOURCE` removed**: was set to empty string in Vercel prod; changed to `mock` then removed entirely to force bundle fallback path

## Key Decisions
- Remove `categorias` from schema entirely since no migration was ever created for it and column never existed in DB
- Graceful degradation instead of crash: auth-edge warns instead of throws; AI insights returns `answer: null` instead of 503
- Use `VERCEL_TOKEN` env var over `--token` flag in CI per Vercel best practices
- Use existing translation keys (`loadError`, `resultsCount`) instead of adding new ones to minimize locale file changes
- Use dynamic imports with outer try-catch in destinations route to isolate module-load failures
